# Plan: Reemplazar WPF-UI NavigationView por WPF estándar

## Problema
El `ui:NavigationView` de WPF-UI causa pantalla negra post-login y crashes. Necesitamos un `.exe` funcional hoy.

## Estrategia
Reemplazar **únicamente** el `NavigationView` en `MainWindow.xaml` por un layout WPF estándar (Grid + ListBox + ContentControl). Las demás vistas (DashboardView, SalesView, etc.) usan controles WPF-UI individuales (Button, TextBox, etc.) que **sí funcionan**, no se tocan.

## Archivos a modificar

### 1. `Views/MainWindow.xaml`
Reemplazar todo el contenido del `<Grid>` por:
```
+-- Grid (2 filas)
    +-- Fila 0: Header (título "GVP", CurrentModuleName, theme toggle, avatar + username + logout)
    +-- Fila 1: Grid (2 columnas)
        +-- Columna 0: StackPanel con botones de navegación estilizados (Dashboard, Ventas, Productos, etc.)
        +-- Columna 1: ContentControl x:Name="MainContentArea" (donde se cargan las vistas)
```
- Sin `ui:NavigationView`, sin `ui:SnackbarPresenter`, sin `ui:ContentDialogHost`
- Los botones del sidebar usan `Command="{Binding NavigateToDashboardCommand}"` etc.
- Mantener theme toggle y logout

### 2. `Views/MainWindow.xaml.cs`
- Eliminar `SnackbarPresenter` y `SnackbarService` (no disponibles sin NavigationView)
- Mantener `OnLoaded` que llama a `_viewModel.Initialize(MainContentArea)`

### 3. `ViewModels/MainWindowViewModel.cs`
- Sin cambios (ya está limpio de código de login)
- `Initialize` navega a `DashboardView`

### 4. `App.xaml.cs`
- Quitar registros DI de `ISnackbarService`, `SnackbarPresenter` 
- Mantener `LoginWindow` + scope como está ahora
- `MainWindow` sigue igual

### 5. `App.xaml`
- Quitar `Wpf.Ui Light.xaml` del merged dictionary (los temas de WPF-UI no se usan más)
- Mantener `LightTheme.xaml`, `ModernBaseStyles.xaml`, y los converters

### 6. `SistemaGVP.WPF.csproj`
- Evaluar si se puede quitar el package `Wpf.Ui` (probablemente no, porque otras vistas lo usan)
- Si otras vistas solo usan controles básicos de WPF-UI (Button, TextBox), mantener el package

## Resultado esperado
- LoginWindow abre → login exitoso → MainWindow abre → Dashboard carga en el ContentControl
- Sidebar funcional con navegación a todas las vistas existentes
- Sin dependencia del `NavigationView` problemático
- El resto del código (servicios, ViewModels, vistas) no se modifica

## NOTA: Si esto falla, plan B
Reemplazar WPF completamente por una app de consola con ASP.NET Core + Kestrel sirviendo Razor Pages como interfaz web local. El backend (Domain, Application, Infrastructure) se reutiliza 100%.
